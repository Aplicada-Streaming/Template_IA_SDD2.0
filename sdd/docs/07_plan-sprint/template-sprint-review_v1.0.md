# 3. Instrucciones de Uso

1. Copiar este template y renombrarlo como `sprint-XX-review_v1.0.md`.
2. Completar la información general antes de la sesión de review.
3. Durante la demo, registrar el estado de cada ítem demostrado.
4. Documentar todo el feedback recibido de los stakeholders.
5. Registrar las métricas y actualizar el documento de [velocidad del equipo](velocidad-equipo_v1.0.md).
6. Confirmar las decisiones tomadas y ajustes al backlog con el Product Owner.
